2.Technical

2.1 General Structure

Here is the structure of the project, as we can see the first tier is based around the types of files which includes css, img and php, it also includes 2 other folders, Testchamber is used to test functionalities before integrating into the main folders and weather is the section used for the accuweather api which includes all functionality present inside.

Inside PHP folder we have 2 folders,

Actions and Forms, forms usually lead into actions as one is the input to the other, inside them you will find folders related to the tables inside the database.

Each of them usually will have all the actions and forms needed for the basic select, insert, update and delete, meanwhile others may have other forms actions which may include ajax functionalities or custom selects.

The php files not in a folder are for miscellaneous purposes that do not correspond to a folder.

2.2 General code structure.

Form:

It will have an include of both the header and footer, and in the middle it will include the form in html tags.

Action:

Currently there are 2 different actions layout, one includes header and footer and can show an output, same as last time the happenings of the code reside in the middle, we also include a connection document that is used to enter the database, the other version removes the header and footer and it includes a header location to move to another page once its action is done.
